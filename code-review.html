Overview

This code review documents my evaluation of selected artifacts for my CS 499 ePortfolio and outlines the enhancements I planned to strengthen my demonstration of core Computer Science program outcomes. The goal of the review was to identify opportunities to improve code quality, algorithmic efficiency, data handling, and overall maintainability. I used the review process to connect my prior coursework to professional software practices, including clear documentation, testing, and secure design thinking.

Artifacts Reviewed

  • SYLTrack (Mobile Application) – selected to represent Software Design & Engineering and Databases through Android development and data persistence.
  • CS 320 Appointment Service – selected to represent Algorithms & Data Structures through structured test design, validation, and data-driven execution patterns.

Planned Enhancements (Summary)

  • Software Design & Engineering (SYLTrack): improve structure, organization, and overall usability; strengthen documentation and maintainability.
  • Algorithms & Data Structures (CS 320 Appointment Service): refactor repetitive unit tests into more efficient data-driven/parameterized tests using structured collections.
  • Databases (SYLTrack): strengthen database design and data handling, with clear persistence logic and reliable CRUD behavior.

Reflection

Completing the code review helped me better understand how to evaluate software as a professional product rather than just a completed assignment. It also showed me the importance of communicating clearly about design decisions and enhancements. The review process made it easier to plan realistic, measurable improvements that demonstrate growth across categories, especially through cleaner structure, stronger testing strategies, and more reliable data management.
